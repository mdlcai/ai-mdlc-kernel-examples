Request flow — ticket creation

How a single POST /v1/tickets moves through the system.

Validate & authorize

Zod parses the body; Bearer JWT → RBAC permission + org scope established in request context.

Persist atomically

One transaction: allocate per-org ticket number (row-locked), insert ticket + SLA state + audit row, all under RLS.

Enqueue async work

Routing + SLA breach-check jobs pushed to BullMQ; ticket.created emitted to the org room.

Worker routes & notifies

Routing worker auto-assigns; notify worker writes an in-app notification + an outbox email intent.

Guaranteed delivery

Outbox relay delivers email/webhooks at-least-once with idempotency keys; UI updates live over WebSocket.
