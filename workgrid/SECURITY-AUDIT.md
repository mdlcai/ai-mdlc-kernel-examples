# SECURITY-AUDIT.md — WorkGrid (build 0.1.3)

Comprehensive-depth narrative audit against SPEC.md §4 (Security Requirements) + ARCHITECTURE.md §9 (Architectural Invariants). 3-pass: initial scan → auto-remediate → re-scan. Every control claimed in SPEC §4 / ARCHITECTURE §9 / REPORT.md was grepped against the actual code and cited `file:line` — not echoed.

## Scope & method
- **Dependency vulnerability:** `npm audit` over the workspace lockfile (`node_modules` + `package-lock.json` present).
- **SAST / manual review:** read auth (`auth.service.ts`), authN/authZ guards, multi-tenant isolation (RLS migration + `TenantContext`/`TenantScopeInterceptor` + per-service repository sourcing across all 9 domain services and both queue workers), input validation, injection/parameterization, secrets handling, rate-limiting, anti-enumeration, webhook HMAC, audit logging.
- **Secret detection:** regex sweep of `apps/**` source (excluding `node_modules`/`.next`) for AWS keys, private keys, Slack/Stripe/GitHub tokens, JWT/session secrets, DB passwords; checked for a production-secret boot assertion.
- **Config / infra:** `docker-compose.yml`, `Dockerfile.api`/`Dockerfile.web`, `deploy/nginx/nginx.conf`, `deploy/postgres/init.sql` (Postgres role model / RLS FORCE), CORS, helmet headers, CSP.
- **Env-var audit:** code-referenced secrets vs `.env.example`; dev-default values for vars that must be secrets in prod.
- **Reconciliation:** cross-checked against the inline audit recorded in `REPORT.md` / `DECISIONS.md` (ADR-0011/0012/0013) — this build dispositioned only the postcss advisory inline and never emitted a standalone file; two claimed controls were found inert that the inline pass missed.

## Pass 2 — auto-remediation
- **#1 FIXED (HIGH):** added `assertProductionSecrets(config)` (`config/configuration.ts`) and called it in `main.ts:20` before `NestFactory.create`. In `NODE_ENV=production` it throws (aborts boot) if `JWT_ACCESS_SECRET`/`JWT_REFRESH_SECRET` are unset, still equal a dev default, or are equal to each other. No-op outside production (dev/test keep the convenience defaults). Mirrors the reference build's `assertProductionSecrets()` remediation.
- **#2 FIXED (MEDIUM):** applied `@Throttle(AUTH_THROTTLE)` at the `AuthController` class level (`auth.controller.ts:17`) — login, register, and refresh now enforce 10/min per tracker (before the service-layer argon2 verify) instead of the lenient global default. Wires the limiter that was defined-but-inert.
- **#3 FIXED (MEDIUM):** set `trust proxy = 1` on the Express instance (`main.ts`, via `getHttpAdapter().getInstance().set(...)`) so the throttler tracker and audit IP key off the real client address behind nginx (which already forwards `X-Forwarded-For`/`X-Real-IP`).
- **#6/#7 ACCEPTED-with-rationale (per triage rule):** build-time-only dev-toolchain CVE — postcss is absent from the runtime container image (`Dockerfile.api`/`Dockerfile.web` ship only compiled output; postcss never runs in the request path) and WorkGrid never stringifies attacker-controlled CSS. Not escalated. The sole upstream "fix" is a Next→v9 downgrade (prohibited framework downgrade). Already recorded as ADR-0013.
- **#4 / #5 NOT auto-remediated — logged as blocking ADR residuals (see Pass 3):** both require an architectural decision absent from ARCHITECTURE.md (an outbound HTTPS call to api.pwnedpasswords.com with SSRF-allowlist + offline-test implications for #4; a key-management / encryption-primitive choice for #5). Per the gate rule "never silently swap/invent a security primitive," these are escalated for human adjudication rather than fabricated.
- `VERSION.md` bumped 0.1.2-functional-verified → **0.1.3** with a history entry.

## Adversarial review highlights (verified controls)
- **Tenant isolation (RLS FORCE):** `0001-init-rls.ts:51-60` runs `ENABLE` + `FORCE ROW LEVEL SECURITY` and a `USING/WITH CHECK (organization_id = current_setting('app.current_tenant_id', true)::uuid)` policy for every `RLS_TABLES` entry. The app role is a **non-superuser** (`deploy/postgres/init.sql:4` `CREATE ROLE workgrid ... NOSUPERUSER`) so FORCE actually constrains it.
- **Tenant scoping is uniform across ALL data paths (verified by grep, not assumed):** all 9 tenant-domain services source repositories exclusively from the GUC-bound transactional manager — `this.tenant.manager.getRepository(...)` in `projects.service.ts:26`, `tasks.service.ts:32`, `tickets.service.ts:33`, `workflows.service.ts:39`, `reports.service.ts:29`, `notifications.service.ts:32`, `dashboard.service.ts:38`, `audit.service.ts:21`, `tickets/sla.service.ts:30`. The GUC is set per request in `tenant-scope.interceptor.ts:38` (`set_config('app.current_tenant_id', $1, true)`) inside a transaction. The only services using the default DataSource are identity/auth (non-tenant tables: users/orgs/memberships/invitations/refresh_tokens) and webhooks (`webhook_delivery`, non-RLS) — all correct. Both queue workers (`outbox-relay.service.ts:144`, `sla-escalation.service.ts:82`) set the tenant GUC on a fresh queryRunner before any RLS-scoped insert.
- **Cross-tenant existence non-leak:** `tenant.guard.ts` and `orgs.service.ts:74` return `NOT_FOUND` (never `FORBIDDEN`) when the user isn't a member / the path id ≠ org context — other tenants' existence does not leak (SEC-1).
- **AuthZ deny-by-default:** global `JwtAuthGuard` → `TenantGuard` → `RbacGuard` chain (`common.module.ts:24-28`); `rbac.guard.ts:36` enforces `ROLE_RANK[org.role] < ROLE_RANK[required]` → 403, and a route needing a role with no resolved org → NOT_FOUND.
- **Password verifier (NIST 800-63B-correct):** argon2id (`auth.service.ts:124`), length-not-composition (no complexity regex anywhere — correct, composition rules are SHALL-NOT). Note: HIBP breach-screen (a SHALL) is the open #4 gap.
- **Anti-enumeration parity:** login runs a real argon2 verify against a constant `DUMMY_HASH` for unknown emails to equalize timing (`auth.service.ts:30,168`); register returns an identically-shaped success and issues **no** tokens when the email already exists (`auth.service.ts:117`, `auth.controller.ts` sets cookies only when `tokens` present). No password-reset endpoint exists, so no sibling-path parity gap there.
- **Inbound webhook HMAC before processing (INV-4):** `webhooks.controller.ts` calls `verifySignature(...)` and throws 403 before `processEvent(...)` for github/slack/resend; verifiers use `timingSafeEqual` with a length-guard and reject on empty secret (`signature.ts:9,33`); Slack enforces a ±300s replay window (`signature.ts:61`). Raw body preserved for HMAC in `main.ts:18`.
- **Idempotency / dedup:** `outbox_event(idempotency_key)` and `webhook_delivery(source, external_id)` UNIQUE constraints asserted in `0001-init-rls.ts:17-19`; webhook replay handled by insert-first + 23505-catch (`webhooks.service.ts:60`).
- **Audit append-only (INV-11):** DB trigger `trg_audit_no_mutate` raises on UPDATE/DELETE of `audit_log` (`0001-init-rls.ts:70-75`); `audit.interceptor.ts` inserts one row per mutating request inside the request transaction; no service issues UPDATE/DELETE on audit_log.
- **Input validation (SEC-11):** global `ValidationPipe({ whitelist: true, forbidNonWhitelisted: true, transform: true })` (`main.ts:37`) rejects unknown fields; all writes go through class-validator DTOs. All raw SQL is parameterized (`$1`-style) — no string-concatenated queries found.
- **Transport / headers (SEC-10):** helmet with HSTS preload (`main.ts:29`), nginx HTTP→HTTPS 301 + HSTS + TLS1.2/1.3 (`nginx.conf`), auth cookies `httpOnly + secure + sameSite` (`auth.controller.ts`), CORS locked to `config.corsOrigins` with credentials.
- **Real-time tenant rooms (INV-15) & structured logging (INV-12) / PII redaction (INV-13):** events emit to tenant rooms only; pino redacts authorization/cookie/password/token/email (`logger.config.ts:14-33`); no `console.log` in `apps/api/src`.
